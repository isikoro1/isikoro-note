## ざっくり

MVC は、アプリケーションを Model、View、Controller に分けて考える設計パターンです。

## 何に使う？

画面表示、入力受付、業務データの扱いを分けることで、コードの役割を整理しやすくします。
Web フレームワークの説明でよく出てきます。

## 自分用メモ

Struts や Spring MVC の理解に必要。まずは「どのコードが何の責務を持つか」を読むための観点として使う。
