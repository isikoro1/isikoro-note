## 概要

Clojure は、Lisp系の構文を持つプログラミング言語です。
JVM上で動作し、Java の資産を使いながら、関数型プログラミング寄りの書き方ができます。

特徴は、オブジェクトの状態を直接書き換えるよりも、値を変換して新しい値を返す考え方を重視する点です。
そのため、状態管理、並行処理、データ変換を整理して扱いやすい言語として見られます。

Java や Python のような一般的な命令型言語に慣れていると、最初は括弧の多さと書き方に違和感があります。
ただし、構文そのものは小さく、慣れると「データをどう変換するか」に集中しやすい言語です。

## 特徴

### Lisp系の構文

Clojure は Lisp 系の言語なので、関数呼び出しを括弧で表します。

```clojure
(+ 1 2)
```

Java や Python では `1 + 2` のように書く処理も、Clojure では「関数名を先頭に置き、その後ろに引数を並べる」形になります。
この書き方は最初は読みにくく感じますが、すべての処理を同じ形で扱えるという強みがあります。

### JVM上で動く

Clojure は JVM 上で動きます。
そのため、Java のライブラリや既存資産を利用できます。

これは、単に新しい言語を学ぶというより、Java エコシステムの上で別の考え方を使うという位置づけに近いです。
業務システムで Java が使われている環境でも、思想として参考にしやすい部分があります。

### 不変データを重視する

Clojure では、データを直接変更するのではなく、新しいデータを作って返す考え方を重視します。

状態を直接書き換える処理が多いと、どこで値が変わったのか追いにくくなります。
不変データを中心にすると、処理の流れを「入力 → 変換 → 出力」として追いやすくなります。

これは、React の state 更新や、関数型プログラミングの考え方ともつながります。

### REPLで試しながら書ける

Clojure は REPL での開発体験が重視されます。
REPL は、コードを少しずつ実行しながら動作を確認できる対話環境です。

ファイルを書いてから全体を実行するだけでなく、関数単位で試し、結果を見ながら開発できます。
小さく試して理解する学習スタイルと相性が良いです。

### マクロで言語を拡張できる

Clojure では、マクロによってコードを生成したり、独自の書き方を作ったりできます。

ただし、これは強力な反面、読みづらいコードを作る原因にもなります。
最初からマクロを使いこなすより、まずは普通の関数、データ構造、変換処理を読む方が現実的です。

## Java と比べた見方

Java は、クラス、オブジェクト、メソッドを中心に設計することが多いです。
Clojure は、データと関数を中心に考えます。

たとえば Java では、状態を持つオブジェクトに対してメソッドを呼ぶ設計になりがちです。
Clojure では、マップやリストのようなデータに対して、関数を組み合わせて変換していく見方が強くなります。

この違いを知ると、Java を書くときにも「状態を持ちすぎていないか」「副作用が広がりすぎていないか」を考えやすくなります。

## 注意

日本語で「クロージャー」と書くと、関数が外側の変数を保持する仕組みである closure と混ざりやすいです。
このページでは、プログラミング言語としての Clojure を扱います。
